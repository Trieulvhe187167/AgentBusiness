Create KB

Create a new Knowledge Base and define who is allowed to access it.

Tên hiển thị của KB trong giao diện quản trị, ví dụ `FAQ tuyển sinh` hoặc `Chính sách nội bộ`.
Mã định danh kỹ thuật của KB, nên ngắn gọn và ổn định, ví dụ `faq_admission` hoặc `internal_policy`.
`Public` cho mọi người, `Internal` cho nhân sự nội bộ, `Admin Only` chỉ cho quản trị viên.
Mô tả ngắn nội dung KB để admin dễ nhận biết phạm vi dữ liệu bên trong.

Google Drive Source

Create a Drive sync source bound to the currently selected Knowledge Base.

Tên dễ nhận biết của nguồn sync, ví dụ `Ops Drive` hoặc `Admissions Docs`.
ID thư mục Google Drive sẽ được quét và import vào KB đang chọn.
Để trống nếu source nằm trong thư mục thường. Chỉ điền khi folder thuộc Shared Drive.
Danh sách pattern phân tách bằng dấu phẩy. Để trống để lấy mọi file hợp lệ.
Pattern loại trừ tên file không muốn import, ví dụ `draft-*` hoặc `tmp-*`.
Drive sources are always bound to the selected KB.
